# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
* 나만의 정기 구독 비용 관리 시스템
<br>


## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 사용자가 자신이 이용 중인 정기 구독 서비스와 월 구독 비용을 관리할 수 있도록 만든 프로그램입니다.
  
* 사용자는 플랫폼 한 개 당 월에 사용할 수 있는 예산을 먼저 입력하고, 이후 자신이 구독 중인 플랫폼 이름과 월 이용금액을 입력할 수 있습니다. 시스템은 입력된 모든 구독 서비스의 월 비용을 합산하여 총 월 구독 비용을 계산합니다. 또한 계산된 총 비용이 사용자가 입력한 플랫폼 월 사용 가능 예산을 초과하는 지 비교하여, 예산 초과 여부를 알려줍니다. 예산을 초과한 경우에는 "사용가능 금액을 초과합니다. 줄여야 합니다." 라고 출력하고 초과하지 않은 경우에는 "사용가능 금액을 초과하지 않습니다." 라고 출력하도록 구성할 계획입니다. 반복문과 메뉴를 이용하여 구독 서비스 추가, 전체 목록 조회, 총 비용 계산, 특정 서비스 삭제 등의 기능을 수행할 수 있습니다. 최종적으로는 리스트를 활용하여 여러 개의 구독 정보를 저장하고, 사용자의 소비 습관을 간단히 관리할 수 있는 프로그램으로 확장할 계획입니다.

<br>


## 3. 예상 기능 및 메뉴 (최소 5개)

1. 월 사용 가능 금액 입력
2. 정기 구독 서비스 추가
3. 전체 구독 서비스 조회
4. 총 월 구독 비용 계산
5. 예산 초과 여부 확인
6. 특정 구독 서비스 삭제
0. 종료

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 시나리오 기획
* 사용자로부터 플랫폼 개당 월 사용 가능 금액, 구독 플랫폼 이름과 월 구독 비용을 입력받습니다.
* 구독 플랫폼 월 구독 비용을 계산하여 총 월 구독 비용과 사용자의 월 사용 가능 금액을 계산 및 출력합니다.
### **🤖 AI 파트너십 과정**

 1. **내용 1 : 파이썬 콘솔 프로그램에 대한 주제 추천**
    * **프롬프트 요약:** ""나는 파이썬 기초를 배우는 대학생이야. 변수/입출력>조건문>반복문/메뉴>함수/배열로 발전시킬 콘솔 프로그램을 만들어야 해. 수업에서 배운 내용을 활용할 수 있어야 하고 어렵지 않은 수준의 새로운 아이디어였음 좋겠어. 프로그램 주제를 5가지 추천해줘.""
    * **적용 내용:** AI가 제안한 여러 아이디어 중 **'OTT 구독 서비스 관리'**라는 컨셉이 재산 관리 측면에서 참신하다고 느껴 채택함. 이를 보고 OTT를 넘어 포괄적으로 프로그램을 만들고 싶어 월 정기 구독 측면에서 프로그램을 생각하게 됨.  
      
 2. **내용 2 : 정기 구독 관리 시스템이 지금까지 배운 파이썬 내용을 잘 반영하는지**
    * **프롬프트 요약:** ""나는 사용자가 정기 구독 서비스 이름과 월 비용을 입력하면 이를 저장하고 관리하는 프로그램인 정기 구독 관리 시스템을 하고 싶어. 저장된 구독들의 월 총 지출을 계산하여 사용자가 설정한 예산과 비교하는 거지. 너가 나의 아이디어를 보고 지금까지 배운 파이썬 내용으로 구현하기 어렵다면, 나의 전체 흐름은 잃지 않게 조금 쉽게 변형해서 보여줘.""
    * **적용 내용:** AI가 충분히 구현 가능하다고 밝힘. AI가 내가 원하는 프로그램에 맞는 시나리오 제목을 나만의 정기 구독 비용 관리 시스템으로 하면 OTT뿐만 아니라 전체 구독 프로그램으로 확장해서 할 수 있어 좋겠다고 추천함. 추천을 따라 시나리오의 제목을 월 구독 비용 관리 시스템에서 나만의 정기 구독 비용 관리 시스템으로 바꿈.
   
 3. **내용 3 : 1차 과제 입출력 형변환 시점에서 이 프로그램이 어떻게 진행될지**
    * **프롬프트 요약:** ""1차 과제 입출력 형변환 시점에서 이 프로그램이 입력받은 데이터 형변환은 어떻게 해야 될지 등 전체적인 진행 과정을 대략적으로 보여줘.""
    * **적용 내용:** AI가 이 프로그램의 대략적 1차 실행 과정은 사용자에게 월 사용 가능 금액, 구독 플랫폼 정보, 각 플랫폼의 월 비용 입력받기와 총 구독 비용 계산이고 예산은 int형, 플랫폼 이름은 string이기에 input으로 입력받길 추천한다 얘기함. 다만, AI가 입력받을 게 많아지면 복잡하니 구독 플랫폼을 3개 정도로 제한하자고 밝힘. 1차 과제에서는 반복문이 사용 안되기에 어쩔 수 없이 입력받는 구독 플랫폼을 3개로 수정하고 사용자가 입력하는 내용을 월 사용 가능 금액에서 플랫폼 개당 월 사용 가능 금액으로 수정함.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
(예시)
  * 이번 과제는 용사 등록 및 정밀 진단 시스템 구현으로, 사용자로부터 데이터를 입력받아 리스트에 저장하고, 가중치 연산을 통해 결과값을 도출하는 시스템을 구축하는 것입니다. 특히 `if-elif` 조건문과 `and/or` 논리 연산자를 활용하여 상황에 맞는 등급과 특별 칭호를 부여하는 지능형 로직을 완성해야 합니다.
  * input()으로 용사의 이름, 체력, 마력, 공격력을 입력받아 리스트에 저장.
  * 가중치 공식을 적용하여 종합 전투력 산출: (체력 * 0.7) + (마력 * 0.5) + (공격력 * 1.8).
  * if-elif-else를 사용하여 S등급부터 F등급까지 5단계 등급 판정 로직 추가.
  * and 연산자를 활용하여 '전설의 대마법사' 등 특별 칭호 부여 기능 구현.
    
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 길드 매니저 컨셉 및 로직 도출**
    * **프롬프트 요약:** "내가 길드 매니저가 되어 용사를 관리하는 프로그램을 만들 거야. 1학년 수준에서 리스트와 조건문을 활용해 등급을 매기는 재미있는 로직을 짜줘."
    * **적용 내용:** AI가 제안한 '가중치 전투력 산출' 방식과 'S등급이면서 마력이 높으면 대마법사 칭호 부여'라는 복합 조건 아이디어를 실제 코드에 반영함.

2. **내용 2: 효율적인 리스트 활용법 자문**
    * **프롬프트 요약:** "파이썬 기초 문법 단계(리스트 → 조건문 → 반복문 → 함수/딕셔너리)에 맞춰, '용사 길드 관리자' 컨셉의 프로그램이 어떻게 발전할 수 있을지 4단계 기능 확장 로드맵을 제안해줘."
    * **적용 내용:** 단순 입출력을 넘어, **'기초 정보 입력 및 리스트 저장(1차) → 전투 등급 판독기 구현(2차) → 무한 메뉴 시스템 구축(3차) → 딕셔너리를 활용한 다중 캐릭터 관리(4차)'**로 이어지는 논리적인 확장 로드맵을 확립함.
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: 숫자를 입력했는데 계산이 안 되는 현상 (TypeError)**
    * **원인:** `input()`으로 받은 데이터는 기본적으로 '글자'라서 숫자와 곱셈 연산이 불가능했음.
    * **해결:** `int()`와 `float()` 함수를 사용하여 입력값을 숫자로 바꾸는 '형변환' 마법을 부려 해결함.
2. **문제 2: 조건문 뒤에 자꾸 생기는 빨간 줄 (SyntaxError)**
    * **원인:** `if`와 `elif` 조건식 끝에 마법의 기호인 `:`(콜론)을 자꾸 빠뜨려서 발생함.
    * **해결:** 파이썬의 모든 조건문과 반복문 끝에는 항상 땡땡(`:`)이 들어가야 한다는 규칙을 학습하고 수정함.
